# IN$DEX Sovereign Workspace Layer — Mission Rooms
**Canonical strategy doc — Session 114 (continued, part 5) — 8 Jul 2026**

---

## One-Sentence Doctrine

Mission Rooms turn citizen intention into organised, trusted, coordinated action inside IN$DEX.

---

## 1. The Crucial Addition

**Sovereign Workspaces** are shared mission hubs inside IN$DEX where citizens collaborate around a business, product, event, community, cultural project, governance proposal, learning path, merchant store, or family legacy.

Each workspace can contain:
- Channels
- SIINDEX bots / agents
- Citizen roles and permissions
- Files and assets
- Marketplace listings
- Governance discussions
- Learning paths
- Treasury or budget notes
- Cultural permission records
- Live commerce events
- Task workflows
- Decision logs
- Reputation contributions
- Opportunity Feed links

**Mission Rooms = the coordination layer of IN$DEX.**

---

## 2. Why This Matters

IN$DEX has strong systems around:
- ImageNation · Opportunity Feed · Sovereign Social · Grid Account
- Wisdom Score · Life Graph · Cultural Protection · Governance
- Marketplace · SIINDEX

But citizens need a place to **organise the work**.

| Citizen type | Why they need a Mission Room |
|---|---|
| Creator launching a product | coordinate creation, listing, media, live commerce |
| Merchant running live commerce | order flow, customer questions, disputes, bundles |
| Community protecting cultural IP | permission requests, elder review, restrictions |
| Governance council | proposals, explainers, voting, decision ledger |
| Child citizen + guardian | safe learning, guardian approvals, earning limits |
| Family building legacy | archive, cultural records, generational memory |

**Without Mission Rooms, IN$DEX ideas scatter. With them, citizens build together.**

---

## 3. Seven Mission Room Types

### 1. Citizen Workspace (Personal)
Personal command centre for one citizen.
Contains: SIINDEX mission briefs, personal goals, income ideas, learning paths, Life Graph, Brain Passport, wallet safety reminders, next best actions.

### 2. Merchant Workspace
For sellers and businesses.
Contains: product listings, customer questions, order updates, disputes, live commerce planning, sales analytics, bundle ideas, trust score improvements, SIINDEX merchant bot.

### 3. Creator Workspace
For creators turning content into ownership.
Contains: content calendar, digital assets, licensing, media kits, product launches, subscriptions, community posts, royalty records, SIINDEX creator bot.

### 4. Community Workspace
For local groups, villages, diaspora, churches, clubs, associations.
Contains: announcements, events, fundraising, community votes, volunteer coordination, cultural protection, emergency support, shared resources.

### 5. Cultural Stewardship Workspace
For elders, families, creators, and cultural guardians.
Contains: cultural asset review, permission requests, attribution rules, commercial approval, revenue-sharing terms, restricted knowledge flags, misuse reports, educational archives.

### 6. Governance Workspace
For proposals, councils, stewards, and citizens.
Contains: proposal discussion, SIINDEX explainer, treasury impact, risk simulation, vote timeline, citizen questions, council notes, decision ledger.

### 7. Child Citizen Workspace
Guardian-controlled safe learning hub.
Contains: learning projects, guardian approvals, earning limits, safe creation tools, school projects, family archive, protected identity, transition plan for 18+.

| SIINDEX Creator Bot | Posts → products, courses, events, licensing, media kits |

**Every agent must have:** role, allowed actions, forbidden actions, data access level, human review requirements, consent requirements, audit log requirements.

**No SIINDEX agent may** move funds, publish content, expose child data, approve cultural commercialisation, or alter governance outcomes without required consent and review.

Citizens create simple automations:

| Trigger | Action |
|---|---|
| New order comes in | notify merchant workspace |
| Product sells out | suggest restock or bundle |
| Governance proposal opens | create citizen explainer |

| Live commerce event ends | generate replay page + follow-up posts |

| Dispute appears | open trust workflow |
| Citizen completes course | update Life Graph + credential |
| Merchant receives 10 good reviews | suggest premium storefront |
| Community event created | build event page + broadcast kit |

Every workflow must show: Trigger → Steps → Responsible roles → Required approvals → Data used → SIINDEX agents involved → Risks → Audit trail → Success signal → Failure signal.

## 6. Sovereign Data Residency

Citizens and communities should know:
- Where their data is stored
- What kind of data is stored
- Who can access it
- Which region applies
- Whether cultural data has special storage rules
- Whether child data is protected separately
- Whether governance data is public/private
- Whether marketplace data is portable
- How to export, correct, or delete data

Planned IN$DEX regions over time: Australia · Pacific region · Japan/Asia · Europe · North America · Custom community vaults for cultural archives.

- Audit trail is visible
- Notifications prioritise useful action, not engagement
- System always uses "citizens," not "users"
