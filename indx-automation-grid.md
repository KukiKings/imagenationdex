## 6. The Automation Formula

Every workflow must be written as:

```
Trigger → SIINDEX Action → Approval Gate → Output → Memory Update → Review Loop
```

**Example:**
- Trigger: New marketplace listing created
- SIINDEX Action: Improve title, description, price, trust checklist, media kit
- Approval Gate: Citizen approves before publishing
- Output: Listing, Opportunity Feed card, media kit
- Memory Update: Business memory and Life Graph updated
- Review Loop: After 7 days, SIINDEX reviews performance and suggests improvement

## 7. The Core Question

Every automation must answer:
1. What useful outcome does this create?
2. Who benefits?
3. What data is used?
4. What approval is required?
5. What could go wrong?
6. What memory should update?
7. What human, guardian, elder, council, or compliance review is needed?
8. Does this increase citizen capability, trust, opportunity, ownership, sovereignty, or legacy?

If the automation does not clearly improve the civilization, reject or redesign it.

## 8. Required Safety Gates

Every automation must check whether it involves:
- money · wallet actions · identity data · child citizens
- cultural assets · governance · legal/compliance claims
- financial outcomes · public publishing · citizen stories
- private files · sensitive personal data

**If yes, add the correct approval gate:**
citizen confirmation · human review · guardian approval · elder/community approval · governance review · compliance review · council approval · manual publish approval
